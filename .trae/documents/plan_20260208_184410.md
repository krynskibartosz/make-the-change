# Plan de Refactoring Avancé (DRY & Architecture)

Ce plan détaille l'exécution des tâches de refactoring architectural avec un haut niveau d'exigence (Tests, JSDoc, Modularité).

## 1. Consolidation de `DataList`
- [ ] **Migration & Modularisation** :
    - Déplacer `apps/web/src/components/ui/data-list.tsx` vers `packages/core`.
    - Refondre la structure pour être modulaire avec des props TypeScript strictes.
    - Documenter l'intégralité de l'API publique avec JSDoc.
- [ ] **Tests Unitaires** :
    - Créer une suite de tests unitaires (Vitest) pour le nouveau composant dans `packages/core`.
    - Atteindre une couverture de code d'au moins 80%.
- [ ] **Intégration** :
    - Mettre à jour tous les imports dans `apps/web` et autres consommateurs.
    - Vérifier le rendu visuel et la rétrocompatibilité.

## 2. Standardisation de `EmptyState`
- [ ] **Analyse & Unification** :
    - Comparer les implémentations `web` et `core` pour identifier les écarts.
    - Enrichir la version `packages/core` avec toutes les fonctionnalités existantes + nouveaux variants (`loading`, `error`, `empty`).
    - Implémenter une API flexible (customisation icônes/messages).
- [ ] **Validation & Nettoyage** :
    - Valider la nouvelle implémentation dans les contextes d'usage actuels.
    - Supprimer définitivement `apps/web/src/components/ui/empty-state.tsx`.

## 3. Suppression des Utilitaires Redondants
- [ ] **Audit & Remplacement** :
    - Vérifier l'équivalence des signatures entre `apps/web/src/lib/utils.ts` et `@make-the-change/core`.
    - Remplacer systématiquement les imports dans tout le codebase.
- [ ] **Vérification** :
    - Exécuter la suite de tests complète pour détecter toute régression.
    - Supprimer le fichier `utils.ts` obsolète.
