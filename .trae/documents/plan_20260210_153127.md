Je te propose un plan en 4 phases pour transformer ton application en un système dynamique gérable via Supabase, sans ajouter de complexité inutile.

###  Phase 1 : Architecture de la Base de Données (Supabase)
Nous allons créer un schéma dédié `cms` pour ne pas polluer tes données métier existantes.

1.  **Création du Schéma et des Tables** :
    *   `cms.menus` : Pour stocker la structure de navigation (Mega Menu).
        *   Colonnes : `slug` (ex: 'main-header'), `structure` (JSONB).
    *   `cms.pages` : Pour stocker le contenu des pages marketing.
        *   Colonnes : `slug` (ex: 'home'), `sections` (JSONB), `seo` (JSONB).
2.  **Sécurité (RLS)** :
    *   Lecture : Publique (tout le monde peut voir le site).
    *   Écriture : Réservée aux admins (rôle `admin` ou `ambassadeur` selon ta logique).

### Phase 2 : Migration des Données (Hardcoded -> DB)
Nous allons extraire les données actuelles du code pour les insérer en base.

1.  **Extraction du Menu** :
    *   Prendre l'objet `megaMenuContent` de `header.tsx` et l'insérer dans `cms.menus` avec le slug `main-header`.
2.  **Extraction de la Home Page** :
    *   Créer une entrée dans `cms.pages` avec le slug `home`.
    *   Convertir les sections statiques (Hero, Bento Grid, Features) en un objet JSON structuré.

### Phase 3 : Logique Backend (Next.js)
Création d'une couche d'abstraction pour récupérer ces contenus proprement.

1.  **Service CMS** (`src/features/cms/cms.service.ts`) :
    *   Fonction `getMenu(slug)` : Récupère et cache la structure du menu.
    *   Fonction `getPageContent(slug)` : Récupère le contenu d'une page spécifique.
2.  **Typage TypeScript** :
    *   Définir les interfaces `MenuStructure` et `PageContent` pour garder l'autocomplétion dans ton code (très important pour ne pas perdre en productivité).

### Phase 4 : Intégration Frontend
Modification des composants pour utiliser les données dynamiques.

1.  **Refonte du Header** :
    *   Modifier `Header.tsx` pour accepter une prop `menuData`.
    *   Mettre à jour `layout.tsx` (côté serveur) pour récupérer le menu et le passer au Header.
2.  **Refonte de la Home Page** :
    *   Dans `page.tsx`, appeler `getPageContent('home')`.
    *   Remplacer les textes et images en dur par les valeurs reçues.
