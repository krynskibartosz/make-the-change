# Génération du "Mega-Prompt" de Migration pour Assistant IA

Je vais concevoir un prompt complet et structuré selon vos 12 critères, destiné à piloter un autre assistant IA pour la migration architecturale `Web` vers `Core`.

## Structure du Prompt Généré

Le prompt sera rédigé au format Markdown pour une lisibilité maximale et couvrira les points suivants, contextualisés spécifiquement pour le projet "Make The Change" :

1.  **Objectif & Périmètre** : Migration des composants "Premium" (`DataList`, `DetailView`, etc.) et des Hooks de `apps/web` vers `packages/core` pour établir une Single Source of Truth.
2.  **Stack Technique (Strict)** : Next.js 15 (App Router), React 19 (RSC), TypeScript 5.9+, Tailwind CSS, Zod.
3.  **Performance** : Exigence de compatibilité "RSC-first" (éviter `use client` inutile), métriques de bundle size.
4.  **Sécurité** : Isolation des clés (Server Actions), validation stricte des props.
5.  **Qualité du Code** : Architecture "Feature-based" dans Core, TSDoc obligatoire, ESLint strict.
6.  **Stratégie de Tests** : Tests unitaires (Vitest) pour les hooks/utils, Stories (Storybook) pour l'UI.
7.  **Livrables Attendus** : Liste précise des fichiers à créer/déplacer dans `packages/core`.
8.  **Jalons** : Découpage en lots logiques (Utils → UI → Forms → Layouts).
9.  **Interfaces** : Définition des APIs des composants (Props) standardisées.
10. **Cas d'Usage** : Scénarios d'intégration dans `apps/web` après migration.
11. **Déploiement** : Procédure de validation dans le monorepo (TurboRepo).
12. **Monitoring** : Logging des erreurs de rendu, gestion des Fallbacks.

## Action
Je vais créer ce contenu dans un fichier nommé `PROMPT_MIGRATION_ARCHITECTE.md` que vous pourrez copier-coller directement.
