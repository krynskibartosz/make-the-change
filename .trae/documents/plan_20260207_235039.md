Je propose de généraliser le pattern "Product Editor" (Traduction + Auto-save + Crop) à l'éditeur de Projets en suivant ces étapes :

### 1. Base de Données (Schema)
*   **Fichier** : `packages/core/src/shared/db/schema.ts`
*   **Action** : Ajouter les colonnes manquantes pour le SEO multilingue dans la table `projects` :
    *   `seo_title_i18n` (jsonb)
    *   `seo_description_i18n` (jsonb)

### 2. Server Actions
*   **Création** : `apps/web/src/app/[locale]/admin/(dashboard)/projects/actions.ts`
*   **But** : Créer une Server Action `updateProjectAction` sécurisée et typée, similaire à celle des produits, pour gérer la sauvegarde partielle (patch) et la revalidation du cache.

### 3. Contrôleur (Logique & State)
*   **Création** : `apps/web/src/app/[locale]/admin/(dashboard)/projects/[id]/components/project-detail-controller.tsx`
*   **Fonctionnalités** :
    *   Intégration de `TranslationProvider` pour gérer le contexte de langue (FR/EN).
    *   Utilisation du hook `useOptimisticAutoSave` pour la sauvegarde automatique.
    *   Gestion de l'état local et de la fusion avec les données en attente.

### 4. Refonte de l'Éditeur (UI)
*   **Fichier** : `apps/web/src/app/[locale]/admin/(dashboard)/projects/[id]/components/project-details-editor.tsx`
*   **Modifications** :
    *   Transition de `react-hook-form` vers un composant totalement contrôlé (props `projectData` + `onFieldChange`) pour s'aligner sur le pattern Produits.
    *   Remplacement des inputs standards par `TranslatableInputControlled` et `TranslatableTextAreaControlled` pour :
        *   Nom
        *   Description courte
        *   Description longue
    *   Ajout de la section **SEO** (Titres & Descriptions traduisibles).
    *   Adaptation de la logique d'upload d'image (Galerie & Crop) pour utiliser `onFieldChange`.

### 5. Intégration dans la Page
*   **Fichier** : `apps/web/src/app/[locale]/admin/(dashboard)/projects/[id]/page.tsx`
*   **Action** :
    *   Mettre à jour la requête DB pour récupérer les champs `*_i18n`.
    *   Remplacer `ProjectEditClient` par le nouveau `ProjectDetailController`.

Cette approche garantit une cohérence totale entre l'admin Produits et Projets, facilitant la maintenance future.
