# Plan de Refactoring UI & Consolidation Monorepo

Ce plan vise à éliminer la duplication de code critique entre `apps/web` et `packages/core` en consolidant les composants UI avancés dans la bibliothèque partagée.

## 1. Migration & Consolidation de `DataList`
*Objectif : Remplacer la version obsolète du Core par la version avancée du Web.*

1.  **Copier** le fichier [`apps/web/src/components/ui/data-list.tsx`](file:///apps/web/src/components/ui/data-list.tsx) vers [`packages/core/src/shared/ui/data-list.tsx`](file:///packages/core/src/shared/ui/data-list.tsx).
2.  **Adapter** les imports dans le nouveau fichier Core (ex: changer `@/lib/utils` pour un import interne ou relatif).
3.  **Vérifier** la compatibilité des props pour s'assurer que les usages existants dans d'autres packages (s'il y en a) ne cassent pas.

## 2. Standardisation de `EmptyState`
*Objectif : Unifier les styles via des variantes au lieu de duplications.*

1.  **Mettre à jour** [`packages/core/src/shared/ui/empty-state.tsx`](file:///packages/core/src/shared/ui/empty-state.tsx) pour ajouter la variante `dashed` (actuellement codée en dur dans Web).
2.  **Supprimer** le composant local [`apps/web/src/components/ui/empty-state.tsx`](file:///apps/web/src/components/ui/empty-state.tsx).
3.  **Refactorer** les usages dans `apps/web` pour importer depuis `@make-the-change/core/ui` avec la prop `variant="dashed"`.

## 3. Nettoyage des Utilitaires (`cn`)
*Objectif : Supprimer la chaîne d'indirection inutile.*

1.  **Supprimer** le fichier [`apps/web/src/lib/utils.ts`](file:///apps/web/src/lib/utils.ts) (qui ne fait que réexporter).
2.  **Mettre à jour** tous les imports dans `apps/web` pour pointer directement vers `@make-the-change/core/shared/utils`.

## 4. Vérification & Nettoyage Final
1.  **Mettre à jour** les imports dans les fichiers consommateurs identifiés (`products-client.tsx`, etc.) pour utiliser les versions du package Core.
2.  **Supprimer** le fichier local [`apps/web/src/components/ui/data-list.tsx`](file:///apps/web/src/components/ui/data-list.tsx).
3.  **Vérifier** que l'application Web compile et s'exécute correctement avec les nouveaux imports.
