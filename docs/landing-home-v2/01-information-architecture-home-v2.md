# Information Architecture — Home V2

## Objectif

Convertir le trafic mobile vers l’exploration des projets via un récit unique:

1. Promesse
2. Clarté
3. Emotion
4. Désir
5. Rétention
6. Preuve sociale
7. Réassurance
8. Action

## Flux section par section

## 1) Hero Header

- But: capter l’attention en moins de 3 secondes.
- Contenu: H1 + sous-titre + CTA unique.
- Contraintes: texte aligné gauche, vidéo/gif assombrie, logo haut gauche.

## 2) Comment ça marche

- But: expliquer l’équation `1€ engagé = 1 point`.
- Contenu: timeline verticale 3 étapes, icônes simples.
- Contraintes: pas de grandes photos, pas de tags décoratifs.

## 3) Projets

- But: déclencher l’empathie et le clic.
- Contenu: carrousel cartes projets + lien “Voir tous” sous le carrousel.
- Contraintes: CTA explicite dans chaque carte.

## 4) Boutique

- But: matérialiser la récompense.
- Contenu: carrousel grandes cartes + lien “Voir toute la boutique” sous le carrousel.
- Contraintes: pas de wishlist, pas de badges, pas de bouton flottant.

## 5) Gamification

- But: montrer la rétention et l’expérience collective.
- Contenu: bento mobile (BioDex, Séries, Tribus) + CTA secondaire.

## 6) Ecosystème

- But: renforcer la crédibilité.
- Contenu: marquee logos monochrome + sous-titre explicatif.

## 7) FAQ

- But: lever les 3 objections finales.
- Contenu: 3 questions en accordéon.

## 8) Final CTA

- But: convertir les indécis en bas de page.
- Contenu: titre, sous-titre, bouton unique, micro-réassurance.

## Règles de sortie

- Pas de blog/actualités sur la home.
- Pas de section additionnelle.
- Pas de CTA concurrent du CTA principal.
